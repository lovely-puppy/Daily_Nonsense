  **Spring怎么解决三级缓存的问题？**
  > Spring 用三级缓存来解决循环依赖，核心思想是把 Bean 的创建过程拆成两步：先创建一个半成品（实例化但未填充属性），把半成品提前暴露出去，等别人用了这个半成品之后再回来填充属性。三级缓存是为了解决有 AOP 代理时的循环依赖，无法解决构造器注入和prototype作用域的循环依赖。
  
  **Java有哪些数据类型？**
  > 八种基本数据类型：byte、short、int、long、float、double、char、boolean
  > 三种引用类型：class、interface、array
  
  - int 和 Integer的区别？
  > int 是基本类型，Integer是包装类型。int不能为null，integer可以为null；int存在栈里，Integer存在堆里；
  
  - Integer缓存池了解吗？
  > Integer 对 -128到127范围内的值做了缓存，这个范围内自动装箱得到的是同一个对象，用==返回true。超出范围就是new了新对象，==比较地址返回false。所以Integer比较时一定要用equals。
  
  - String是基本类型吗？
  > String 是引用类型。且有字符串常量池，String是不可变对象（用final修饰），每次修改都会创建新的对象。频繁拼接字符串要用StringBuilder。
  
  - ==和equals有什么区别？
  > == : 基本类型比较值，引用类型比较内存地址
  > equals：默认比较地址，但是String、Integer等类重写了equals的方法，改成了比较值
  
  **什么是自动装箱/拆箱？**
  > 装箱就是把基本数据类型转换为包装类型
  > 拆箱就是把包装类型转换为基本数据类型，自动拆箱的时候要注意判空
  
  **面向对象编程有什么特性？**
  > 封装、继承、多态
  > 封装：对外隐藏内部实现细节，只暴露必要的接口。
  > 继承：子类复用父类的属性和方法，在此基础上扩展自己的功能。
  > 多态：同一个方法，不同对象调用有不同的表现。父类引用指向子类对象，运行时根据实际对象类型决定调用哪个方法。
  
  **接口和抽象类有什么区别？**
  > 抽象类更多地是用来为多个相关的类提供一个共同的基础框架，包括状态的初始化，而接口则是定义一套行为标准，让不同的类可以实现同一接口，提供行为的多样化实现。
  
  **为什么重写equals时必须重写hashcode方法？**
  > 因为 HashMap、HashSet 这些哈希集合的工作机制是先用 hashCode 定位桶，再用 equals 比较值。如果只重写 equals 不重写 hashCode，两个 equals 相等的对象可能有不同的hashCode，被分到不同的桶里，equals 根本没机会执行，导致 HashMap 找不到数据、HashSet 无法正确去重。所以 Java 规定 equals 相等的两个对象 hashCode 必须相同，重写 equals 时必须同时重写hashCode，保证两者的一致性。实际开发中一般用 Objects.hash() 方法，把参与 equals 比较的字段都传进去生成 hashCode。
  
  **说说深拷贝和浅拷贝的区别？**
  > 浅拷贝：复制对象本身，但内部引用的对象不复制，共用同一个。
  > 深拷贝：复制对象本身，但内部引用的对象也全部复制一份，完全独立。
  > 三种深拷贝的方法：1、手动递归拷贝；2、序列化反序列化；3、JSON转换；
  
  **String、StringBuilder、StringBuffer有什么区别？**
  > String 是不可变的，底层是 final char 数组，每次修改都创建新对象，循环拼接性能很差。StringBuilder 和 StringBuffer 都是可变的，在原对象上直接修改。区别是 StringBuffer 的方法加了 synchronized 保证线程安全但有性能开销，StringBuilder 没有锁速度更快但线程不安全。实际开发中 99% 的场景用 StringBuilder，因为字符串拼接通常是局部变量不涉及多线程。String 不可变的好处是安全、可以复用常量池、hashCode 可以缓存、天然线程安全。
